# Мобильное тестирование

Проверять опубликованную GitHub Pages-версию, а не локальный `file://`.

| Устройство | Браузер | Минимум |
| --- | --- | --- |
| iPhone | Safari | iOS 16+ |
| iPad | Safari | iPadOS 16+ |
| Android | Chrome | Android 10+ |

Для каждого устройства:

1. Открыть сайт, дождаться загрузки каталога и при ошибке проверить кнопку «Повторить загрузку».
2. Найти товар по артикулу и по названию, выбрать его касанием.
3. Ввести площадь с запятой, выбрать 10% запаса, затем указать произвольный процент.
4. Проверить расчёт, предупреждение о недостатке склада и полный текст из кнопки «Скопировать результат».
5. Проверить, что поля не вызывают масштабирование страницы, а элементы не перекрыты и доступны без горизонтальной прокрутки.
